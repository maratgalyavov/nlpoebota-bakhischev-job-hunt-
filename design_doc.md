# 📄 Design-Doc: HR Career Assistant

## 1. Контекст проекта

### 1.1 Бизнес-задача

| Проблема | Решение | Критерии успеха |
|----------|---------|-----------------|
| Соискатели тратят часы на поиск подходящих вакансий, написание резюме и сопроводительных писем. Многие не знают, каких навыков не хватает для желаемой позиции. | HR-агент проводит «интервью», создаёт резюме, подбирает пдходящие вакансии, помогает с сопроводительным письмом и подчеркивает существующие skill gaps | • 70% пользователей оценивают подборку вакансий как релевантную <br>  |

**Почему это имеет смысл:**
- Без бота: ручной поиск, шаблонные резюме, непонимание требований рынка.
- С ботом: персонализированный подбор, автоматизация рутины, обратная связь по навыкам.
- **Этика**: Бот не гарантирует трудоустройство, а помогает с поиском и подготовкой.

### 1.2 Целевая аудитория и пользователи

| Параметр | Описание |
|----------|----------|
| **Пользователи** | Соискатели (студенты, джуны, мидлы), люди в поиске работы, карьерные переходы |
| **Сценарии** | Текстовое общение в Telegram-боте |
| **Нагрузка** | Пиковая: вечер будней, выходные (до 500 сессий/день) |
| **География** | Россия |

### 1.3 Ограничения и допущения

| ❌ Не будет | ✅ Допущения |
|-------------|--------------|
| Гарантия трудоустройства | Пользователь понимает, что это помощник, а не рекрутер |
| Работа с чувствительными ПДн (паспорт, ИНН) | Данные хранятся локально, не передаются третьим лицам |
| Интеграция с HR-системами компаний | Используем публичные API job-бордов |
| Юридические консультации по ТК | Бот не даёт юридических советов |

## 2. Архитектура решения

### 2.1 Общая схема

```
┌─────────────┐     ┌──────────┐     ┌─────────────┐     ┌──────────────┐     ┌──────────┐
│   User      │ ──→ │ Telegram │ ──→ │  Backend    │ ──→ │  Interview   │ ──→ │   LLM    │
│ (Telegram)  │     │   Bot    │     │ (FastAPI)   │     │   Module     │     │ (Resume) │
└─────────────┘     └──────────┘     └─────────────┘     └──────────────┘     └──────────┘
                                              │
                                              ▼
                                       ┌─────────────┐
                                       │   User      │
                                       │  Embedding  │
                                       │  (Vector)   │
                                       └─────────────┘
                                              │
                                              ▼
                                       ┌─────────────┐     ┌──────────────┐     ┌──────────┐
                                       │ Vector DB   │ ──→ │   Matching   │ ──→ │ Vacancies│
                                       │ (FAISS)     │     │  (k-NN)      │     │  + Gaps  │
                                       └─────────────┘     └──────────────┘     └──────────┘
                                              │
                                              ▼
                                       ┌─────────────┐
                                       │   SQLite    │
                                       │   (логи)    │
                                       └─────────────┘
```

### Логика запроса

1. Пользователь проходит интервью.
2. Ответы агрегируются в structured profile.
3. Генерируется резюме
4. Создается Embedding профиля
5. Выполняется kNN-поиск по базе вакансий.
6. Пользователь выбирает вакансию.
7. Генерируется Cover letter и Skill gap анализ

| Компонент | Роль | Технологии |
|-----------|------|------------|
| Telegram Bot | Интерфейс для пользователя | python-telegram-bot |
| Backend | Обработка запросов, оркестрация | FastAPI + Python |
| Interview Module | Проведение «интервью», сбор данных | State machine, LLM |
| Embedding Generator | Векторизация профиля пользователя | sentence-transformers |
| Vector DB | Хранение эмбеддингов вакансий | FAISS / ChromaDB |
| Matching Engine | Поиск похожих вакансий (k-NN) | Cosine similarity |
| LLM | Генерация резюме, писем, skill gaps | Qwen3.5-Flash |
| SQLite | Хранение пользовательских данных | Локальная БД |

Обработка пользовательских запросов - в порядке очереди

### 2.2 Хранилище знаний / документооборот

## 3. Данные и качество знаний

### 3.1 Сбор и предобработка данных

#### Данные вакансий

| Форматы | Источники |
|---------|-----------|
| JSON, CSV | HH.ru API, Kaggle datasets |

**Шаги предобработки:**
1. Очистка от спецсимволов
2. Обработка зарплат: от/до
3. Извлечение навыков и требований (regex + NER)
4. Токенизация
5. Создание эмбеддингов вакансии

#### Данные пользователя

| Формат | Источник |
|---------|-----------|
| JSON | Интервью |

**Шаги предобработки:**
1. Конкатенация текста или structured-to-text шаблон
4. Токенизация
5. Создание эмбеддингов профиля пользователя 

### 3.2 Векторизация и индексирование

| Параметр | Значение |
|----------|----------|
| Embedding модель | например, `sentence-transformers/paraphrase-multilingual-mpnet-base-v2` |
| Размерность | 768 |
| Индекс | FAISS (локально) / ChromaDB |
| Метрика | Cosine similarity |
| Обновление | Инкрементальное (новые вакансии ежедневно) |


### 3.3 Метрики качества знаний

| Метрика | Как измеряем | Цель |
|---------|--------------|------|
| Покрытие | % категорий вакансий в базе | ≥80% (IT, маркетинг, дизайн, etc.) |
| Актуальность | % вакансий моложе 30 дней и % активных вакансий | ≥70% |
| Консистентность | Нет дубликатов вакансий | < 1% дублей |
| Проверка | Ручная оценка 50 подборок перед релизом | ≥85% релевантных |

## 4. Модель и генерация

### 4.1 Выбор LLM и промптинг

| Параметр | Значение |
|----------|----------|
| Модель | `Qwen3.5-Flash` (open-source) |
| Промпт (резюме) | *«На основе ответов пользователя создай структурированное резюме. Включи: опыт, навыки, образование, проекты, прочую информацию.»* Далее в промпт добавляются ответы пользователя в формате вопрос-ответ. |
| Промпт (письмо) | *«Составь сопроводительное письмо для вакансии {vacancy_description}. Используй резюме {cv}. Тон: профессиональный, 150-200 слов.»* |
| Промпт (gaps) | *«Сравни навыки пользователя {cv} с требованиями вакансии {vacancy_description}. Укажи недостающие навыки и ресурсы для обучения.»* |
| Ограничения | Макс 1000 токенов на ответ, генерация < 15 сек |

### 4.2 Контроль качества ответов

| Метрика | Как измеряем |
|---------|--------------|
| Релевантность вакансий | Оценка пользователей 👍/👎 |
| Качество резюме | Ручная проверка (чек-лист: опыт, навыки, образование) |
| Hallucinations | Фильтр на выдуманные навыки/опыт |

**Механизмы:**
- Если confidence < 0.6 → «Нужно уточнить ответы»
- Fallback на шаблон резюме при ошибке LLM
- Кнопка «Перегенерировать» для альтернатив

### 4.3 Обучение/дообучение

| Аспект | План |
|--------|------|
| Fine-tuning | Не планируется на MVP |
| Few-shot | 3-5 примеров резюме/сопроводительных писем в промпте |
| Версионирование | Промпты в git, логирование версий |
| RLHF | Возможно на фазе сбора фидбэка |

## 5. UX / пользовательский опыт

### 5.1 Сценарии взаимодействия

| Сценарий | Описание |
|----------|----------|
| ✅ Onboarding | Приветствие → объяснение функционала → старт интервью |
| ✅ Интервью | 10-15 вопросов (опыт, навыки, желаемая роль, локация, зарплата) |
| ✅ Подбор вакансий | Показ топ-5 вакансий с % match + ссылки и описание |
| ✅ Резюме | Генерация текста резюме + возможность повторной генерации с комментариями пользователя (но не более X раз) |
| ✅ Сопроводительное | Генерация письма под выбранную вакансию + возможность повторной генерации с комментариями пользователя (но не более X раз) |
| ✅ Skill Gaps | Список недостающих навыков |
| ❌ Нет вакансий | «Пока нет точных совпадений. Расширим поиск?» |
| ❌ Баг | «Что-то пошло не так... Попробуй через 5 минут» |

### 5.2 Диалоговая логика

| Аспект | Решение |
|--------|---------|
| Контекст | Хранение всей сессии интервью в памяти (SQLite) |
| История | Сохранение профиля пользователя (анонимно, по ID Telegram) |
| Тон | Дружелюбный, поддерживающий, профессиональный |
| Язык | Русский |

**Пример потока интервью:**
```
1. Подробно расскажи про свой опыт работы (сколько лет / какие позиции / какой вклад)
2. Какими инструментами владеешь?
3. Какое образование?
4. Какую роль ищешь?
5. Желаемая зарплата?
6. Локация (remote / офис / гибрид)?
7. Полная / частичная занятость?
7. Основные личностные качества
```

### 5.3 Метрики UX

| Метрика | Как собираем | Цель |
|---------|--------------|------|
| Время до первого ответа | Логирование latency | < 3 сек |
| Завершение интервью | % дошедших до конца | ≥60% |
| Возвраты | % пользователей с >2 сессиями в течение n мес. | ≥40% |
| Рейтинг подробок вакансий | Кнопки 👍/👎 | ≥70% отзывов положительные |
| Логирование | Все диалоги для анализа | 100% |

## 6. Безопасность, соответствие и этика

| Аспект | Решение |
|--------|---------|
| Персональные данные | Хранение локально, не передаём третьим лицам |
| Чувствительная информация | Не запрашиваем паспорт, ИНН, точный адрес |

| Нормативы | 152-ФЗ о персональных данных |
| Дисклеймер | «Проверяйте информацию перед откликом» |

## 8. Риски и допущения

| Риск | Вероятность | Влияние | Mitigation |
|------|-------------|---------|------------|
| Галлюцинации относительно навыков/опыта | Средняя | Высокое | Более строгие промпты |
| Мало вакансий в базе | Низкая | Среднее | Парсинг нескольких источников (HH, Habr, LinkedIn) |
| Пользователи вводят ложные данные | Средняя | Низкое | Бот не проверяет, но предупреждает о честности |
| Низкая релевантность       | Средняя     | Высокое | Улучшение представления профиля |
